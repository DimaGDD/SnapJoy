# Архитектура игрового проекта

## Версия движка - 6000.2.6f2

## Общая архитектура

Проект построен на **модульной архитектуре**, где каждый скрипт отвечает только за свою конкретную задачу.

### Пример модулей игрока:
- `InputHandler` — обработка пользовательского ввода
- `Movement` — передвижение персонажа
- `Look` — управление камерой
- `States` — состояния игрока
- `Interact` — взаимодействие
- `и тд`

---

## Архитектурные паттерны

### 🔷 ScriptableObjects
Настройки объектов вынесены в `ScriptableObject`, что позволяет:
- Удобно редактировать параметры
- Создавать несколько конфигураций для разных объектов
- Отделять данные от логики

### 🔷 Singleton (Одиночка)
Используется для глобального управления состояниями:
- `UIManager` — управление UI
- `DialogueManager` — управление диалогами

### 🔷 Observer Pattern (Наблюдатель)
Реализован в `InputHandler`:
- Скрипт обрабатывает ввод
- Рассылает события подписанным методам через статические события

### 🔷 State Pattern (Состояние) - упрощенный
Состояния игрока:
- Отдельный скрипт с состояниями
- Логика поведения зависит от текущего состояния

### 🔷 Наследование и инкапсуляция
- Все предметы наследуются от базовых классов
  - `DefaultPickupItem`
  - `DefaultPushItem`
  - `DefaultHoldItem`
  - `DefaultHolderItem`
  - `DefaultOpenItem`
  - `и тд`
- Каждый предмет может расширять свою логику
- Поля классов защищены от внешних изменений (доступ только через предусмотренные методы и свойства)

## Игровые фичи

### 1️⃣ Передвижение, осмотр, взаимодействие
- Отдельные скрипты для каждой механики
- Используется **New Input System** (Input Actions)
- Каждому действию назначена своя кнопка

### 2️⃣ Подбираемые предметы (`DefaultPickupItem`)
- Предметы, которые можно взять в руку
- Возможность осмотреть и покрутить
- Использование для взаимодействия с другими объектами

### 3️⃣ Толкаемые предметы (`DefaultPushItem`)
- Предметы, которые можно толкнуть при взаимодействии

### 4️⃣ Открываемые предметы (`DefaultOpenItem`)
- Открываются определенным предметом (указывается в инспекторе)

### 5️⃣ Удерживаемые предметы (`DefaultHoldItem`)
- Требуют удержания кнопки взаимодействия

### 6️⃣ Связанные предметы (`DefaultLinkedMover`)
- Связаны с `DefaultHoldItem`
- Реагируют на взаимодействие с удерживаемым предметом

### 7️⃣ Предметы-хранилища (`DefaultHolderItem`)
- Можно положить `DefaultPickupItem`
- Можно забрать предмет обратно

### 8️⃣ NPC (`DefaultNPC`)
- Персонаж для взаимодействия

### 9️⃣ Диалоговая система (`DialogueSystem`)
- Хранение диалогов NPC
- Поддержка ветвлений и вариантов ответов
- Диалоги хранятся в `ScriptableObject`
- Воспроизведением управляет `DialogueManager` (Singleton)
